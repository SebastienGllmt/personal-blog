What is a hash function?

A ten-minute primer on one of computing's most useful primitives.

The core idea

A hash function is a one-way mapping from any input — a word, a file, an entire movie — to a fixed-size fingerprint.

Property 1 — Deterministic

The same input always produces the same output. Hash the string "hello" today, hash it again next year, and you'll get the exact same fingerprint.

Property 2 — Avalanche

Change a single bit of the input and the output looks completely different. There is no smooth gradient between similar inputs.

Where you've already met hashes

Passwords

Sites don't store passwords directly — they store hashes. A leaked database is much harder to abuse when the attacker only has fingerprints.

File integrity

Downloading a Linux ISO? The site publishes a SHA-256 checksum. Hash your local copy and compare — if it matches, the file arrived intact.

Blockchains

Each block is named by its hash, and references the previous block's hash. Tampering with any block invalidates every block that follows. Bitcoin wallet addresses, for instance, are derived by running RIPEMD-160 on top of SHA-256.

That's hashing in a nutshell. Thanks for listening — and happy hashing!
